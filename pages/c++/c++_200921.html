类的继承

问题背景

数据结构实验课上，老师给我们复习复习c++，经过了这个有史以来最长的假期， 我脑中的c++已经从近看变成了远眺了。而第一次实验课就是重要的继承啦！ 第一堂实验课，不知道为何如此之困，小小的打了会瞌睡，醒来之后老师就叫我们写出符合要求的函数了。 刚开始还觉得能吃老本，随便写写不都写出来了吗/滑稽

可是啊，上学期我就没领会到继承的真谛啊。

错误之路

问题是这样的，有一个employee1类，一个company1类，一个bestpaid的main函数， 调用company1类的函数接受姓名，收入的输入并记录到employee1类的对象中。而第二阶段就是由两个基类派生出 employee2和company2两个类，调用company2的函数接受姓名，工作时长以及月薪的输入，最后打印出bestpaid的employee。 听起来很简单，改改不就好了吗。所以我就开始了“修改”之路

employee2.h

#ifndef EMPLOYEE2
#define EMPLOYEE2

#include "employee1.h"

class Employee2:public Employee
{
private:
	int month;
	double monthlySalary;

public:
	Employee2();
	void input();
	bool compare(const Employee2& otherEmp) const;
	
	void copy(const Employee2& otherEmp);
	
	void output() const;
	
};

#endif

改啊改啊，把employee2里加了month，monthlySalary这些成员，还把input()，compare()这些函数也全部重载了 发现执行的还是e1的input()。。。又到company2中各种重写。。。还到处是“未定义的标识符”，又去include其他头文件。。。 最终，在经过3个小时折腾后，程序终于跑起来了！
